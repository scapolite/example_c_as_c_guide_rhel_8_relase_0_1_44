## /rule

Configure auditd flush priority

## /rationale

Audit
data should be synchronously written to disk to ensure log integrity.
These parameters assure that all audit event data is fully synchronized
with the log files on the disk.

## /description

The
`auditd` service can be configured to synchronously write audit event
data to disk. Add or correct the following line in
`/etc/audit/auditd.conf` to ensure that audit event data is fully
synchronized with the log files on the disk:

``` {html="http://www.w3.org/1999/xhtml" ns0="http://checklists.nist.gov/xccdf/1.1"}
flush = 
```
