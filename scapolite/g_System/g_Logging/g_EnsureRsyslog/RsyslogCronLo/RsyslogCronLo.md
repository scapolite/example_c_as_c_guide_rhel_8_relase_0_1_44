## /rule

Ensure cron Is Logging To Rsyslog

## /rationale

Cron
logging can be used to trace the successful or unsuccessful execution of
cron jobs. It can also be used to spot intrusions into the use of the
cron facility by unauthorized and malicious users.

## /description

Cron
logging must be implemented to spot intrusions or trace cron job status.
If `cron` is not logging to `rsyslog`, it can be implemented by adding
the following to the *RULES* section of `/etc/rsyslog.conf`:

``` 
cron.*                                                  /var/log/cron
```
