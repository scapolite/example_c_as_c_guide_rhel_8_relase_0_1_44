## /rule

Ensure rsyslog Does Not Accept Remote Messages Unless Acting As Log
Server

## /rationale

Any
process which receives messages from the network incurs some risk of
receiving malicious messages. This risk can be eliminated for rsyslog by
configuring it not to listen on the network.

## /description

The
`rsyslog` daemon should not accept remote messages unless the system
acts as a log server. To ensure that it is not listening on the network,
ensure the following lines are *not* found in `/etc/rsyslog.conf`:

``` 
$ModLoad imtcp
$InputTCPServerRun port
$ModLoad imudp
$UDPServerRun port
$ModLoad imrelp
$InputRELPServerRun port
```
