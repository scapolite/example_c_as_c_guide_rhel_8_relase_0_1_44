## /rule

Set Default firewalld Zone for Incoming Packets

## /rationale

In
`firewalld` the default zone is applied only after all the applicable
rules in the table are examined for a match. Setting the default zone to
`drop` implements proper design for a firewall, i.e. any packets which
are not explicitly permitted should not be accepted.

## /description

To
set the default zone to `drop` for the built-in default zone which
processes incoming IPv4 and IPv6 packets, modify the following line in
`/etc/firewalld/firewalld.conf` to be:

``` 
DefaultZone=drop
```
