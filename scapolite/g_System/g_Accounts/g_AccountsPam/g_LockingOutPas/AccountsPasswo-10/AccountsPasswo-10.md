## /rule

Configure the root Account for Failed Password Attempts

## /rationale

By
limiting the number of failed logon attempts, the risk of unauthorized
system access via user password guessing, otherwise known as
brute-forcing, is reduced. Limits are imposed by locking the account.

## /description

To
configure the system to lock out the `root` account after a number of
incorrect login attempts using `pam_faillock.so`, modify the content of
both `/etc/pam.d/system-auth` and `/etc/pam.d/password-auth` as
follows:  
  

-   Modify the following line in the `AUTH` section to add
    `even_deny_root`:

        auth required pam_faillock.so preauth silent even_deny_root deny= unlock_time= fail_interval=

-   Modify the following line in the `AUTH` section to add
    `even_deny_root`:

        auth [default=die] pam_faillock.so authfail even_deny_root deny= unlock_time= fail_interval=
