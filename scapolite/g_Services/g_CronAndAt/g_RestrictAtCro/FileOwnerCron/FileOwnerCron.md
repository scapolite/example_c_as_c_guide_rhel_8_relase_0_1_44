## /rule

Verify User Who Owns /etc/cron.allow file

## /rationale

If
the owner of the cron.allow file is not set to root, the possibility
exists for an unauthorized user to view or edit sensitive information.

## /description

If
`/etc/cron.allow` exists, it must be owned by `root`. To properly set
the owner of `/etc/cron.allow`, run the command:

``` 
$ sudo chown root /etc/cron.allow 
```
