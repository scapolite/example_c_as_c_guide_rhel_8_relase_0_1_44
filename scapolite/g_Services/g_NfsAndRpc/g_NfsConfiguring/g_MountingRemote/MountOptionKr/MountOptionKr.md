## /rule

Mount Remote Filesystems with Kerberos Security

## /rationale

When
an NFS server is configured to use AUTH\_SYS a selected userid and
groupid are used to handle requests from the remote user. The userid and
groupid could mistakenly or maliciously be set incorrectly. The
AUTH\_GSS method of authentication uses certificates on the server and
client systems to more securely authenticate the remote mount request.

## /description

Add
the `sec=krb5:krb5i:krb5p` option to the fourth column of `/etc/fstab`
for the line which controls mounting of any NFS mounts.
