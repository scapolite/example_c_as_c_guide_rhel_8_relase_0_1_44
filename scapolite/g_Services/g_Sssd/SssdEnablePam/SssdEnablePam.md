## /rule

Configure PAM in SSSD Services

## /rationale

Using
an authentication device, such as a CAC or token that is separate from
the information system, ensures that even if the information system is
compromised, that compromise will not affect credentials stored on the
authentication device.

## /description

SSSD
should be configured to run SSSD `pam` services. To configure SSSD to
known SSH hosts, add `pam` to `services` under the `[sssd]` section in
`/etc/sssd/sssd.conf`. For example:

``` 
[sssd]
services = sudo, autofs, pam
```
